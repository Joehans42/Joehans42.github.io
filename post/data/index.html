The Data

December 7, 2021

Introduction

The data in this project consists of two separate data sets. The first data set is a list of Wikipedia articles. The Wikipedia articles can be different depending on the choosing of an initial starting page. In this data set however, the initial starting page is pre-selected as the Wikipedia page for Albert Einstein. The first data set is thus a network consisting of the articles that can be reached by clicking the links on the pages within 2 clicks. The Wikipedia page for Albert Einstein alone has over 600 links to other articles and adding up the links on those articles gives a total of 65315 pages. These pages make up the nodes of the network and they have 98825 edges between them.

The second data set is a data set from the Stanford Network Analysis Project (SNAP). The data set contains human navigation paths on Wikipedia which have been obtained through the Wikispeedia game. In Wikispeedia, the objective is to navigate from a source article to a target article by clicking links on the pages. Here they have used a condensed version of Wikipedia where only 4604 articles are used. SNAP has provided both the full navigation paths for these speedruns as well as the full plaintext of each article. Source: http://snap.stanford.edu/data/wikispeedia.html

Motivation

So what is actually interesting about these data sets you might ask? Well, looking at the SNAP data set, we can get an idea of how real humans navigate online data hubs, and looking at the first data set, which we will call Base Wikipedia here, we can look at the actual shortest paths between the nodes. So, where do they differ? Are there more general routes that humans prefer to take, since they cannot know every possible path between pages? That is what we want to find out in this project. What pages are commonly clicked in order to reach a given target?
